# 3. Знакомства и настройка базы данных SQLite 🗄️

Давай подружим наш сервер с базой данных. Представь записную книжку, куда ты быстро заносишь задачки — это и есть SQLite: лёгкая, быстрая и не требует отдельного сервера.

## Что такое SQLite? 📚

SQLite — это встроенная база данных в одном файле. Идеальна для первых проектов: ничего запускать отдельно не нужно, просто подключаем библиотеку и работаем.

## Шаг 1: Устанавливаем зависимость 📦

```bash
npm i sqlite3
```

После установки у тебя в папке `node_modules/` появятся новые пакеты, а в `package.json` — появится запись о новой зависимости.

## Шаг 2: Создаём и инициализируем базу 🧩

Добавим файл `db.js` в папку `database` проекта. Он создаст файл базы (если его ещё нет) и таблицу `tasks`.

```javascript
// Подключаем библиотеки sqlite3 и path
const sqlite3 = require('sqlite3').verbose();
const path = require('path');

// Храним базу данных рядом с db.js, файл будет называться db.sqlite
const db = new sqlite3.Database(path.join(__dirname, 'db.sqlite'));

// Создаём таблицу задач, если её ещё нет
db.serialize(() => {
  db.run(`
    CREATE TABLE IF NOT EXISTS tasks (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      title TEXT NOT NULL,
      done INTEGER NOT NULL DEFAULT 0,
      created_at TEXT NOT NULL DEFAULT (datetime('now'))
    )
  `);
});

module.exports = db;
```

Что мы сделали:
- `id` — уникальный номер задачи
- `title` — текст задачи
- `done` — флаг выполнения (0 или 1)
- `created_at` — когда задача была создана

## Шаг 3: Подключаем базу в сервер 🔗

Открой `app.js` и подключи базу данных. Затем добавь маршрут для создания задачи.

```javascript
// Подключаем БД (добавь рядом с другими require)
const db = require('./database/db');

// Создание задачи (POST /tasks)
app.post('/tasks', (req, res) => {
  const { title } = req.body;

  if (!title || typeof title !== 'string' || title.trim().length === 0) {
    return res.status(400).json({ error: 'Укажи непустой title' });
  }

  const sql = 'INSERT INTO tasks (title) VALUES (?)';
  db.run(sql, [title.trim()], function (err) {
    if (err) {
      return res.status(500).json({ error: 'Не удалось создать задачу' });
    }

    // this.lastID — id добавленной записи
    res.status(201).json({ id: this.lastID, title: title.trim(), done: 0 });
  });
});

// (Не обязательно, но удобно для проверки) Список задач (GET /tasks)
app.get('/tasks', (req, res) => {
  db.all('SELECT id, title, done, created_at FROM tasks ORDER BY id DESC', [], (err, rows) => {
    if (err) return res.status(500).json({ error: 'Не удалось получить список задач' });
    res.json(rows);
  });
});
```

Убедись, что эти маршруты добавлены до `app.listen()`.

## Шаг 5: Аккуратно закрываем соединение с базой 🧹

Чтобы всё работало стабильно, базу нужно «правильно» закрывать, когда ты останавливаешь сервер.

Почему это важно:
- Ресурсы не резиновые: открытая база держит «дверь приоткрытой» и ест память.
- Лимиты: если никогда не закрывать, рано или поздно «закончатся места для новых гостей».
- SQLite — это файл. Если не закрыть, файл может остаться «занятым», и другим операциям будет сложно.

Как сделать правильно (в `app.js`):

```javascript
// Уже есть: подключаем БД
const db = require('./database/db');

// после app.listen()

// Правильно закрываем базу при остановке сервера
const shutdown = () => {
  console.log('Останавливаем сервер... ⛔');
  db.close(err => {
    if (err) {
      console.error('Не получилось закрыть базу:', err.message);
      process.exit(1);
    }
    console.log('База закрыта. До встречи! 👋');
    process.exit(0);
  });
}

process.on('SIGINT', shutdown);  // Ctrl+C в терминале
process.on('SIGTERM', shutdown); // остановка из системных инструментов/хостинга
```

Проверь:
1. Запусти сервер: `npm run dev`.
2. Нажми `Ctrl+C` — в консоли увидишь сообщения «Останавливаем сервер…», затем «База закрыта…».
