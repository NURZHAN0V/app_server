## 4. Регистрация и авторизация пользователей 🔐

В этой главе ты добавишь к своему серверу простую регистрацию, вход по токену (JWT), защиту маршрутов и аккуратную обработку ошибок. Всё по‑простому и без сложных терминов.

---

### Шаг 1. Ставим нужные пакеты 📦

Установим две библиотеки:

- `bcryptjs` — шифрует (хэширует) пароли, чтобы они не лежали в базе «как есть»
- `jsonwebtoken` — создаёт и проверяет токены доступа (JWT)

```bash
npm i bcryptjs jsonwebtoken
```

---

### Шаг 2. Добавим таблицу пользователей в базу 🗄️

Открой файл `database/db.js` и добавь создание таблицы `users`. Она будет хранить почту и «хэш» пароля.

```javascript
// Добавь внутри db.serialize(...)
db.run(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
  )
`);
```

Поля:
- `email` — логин пользователя (делаем `UNIQUE`, чтобы нельзя было зарегистрировать два одинаковых адреса)
- `password_hash` — зашифрованный пароль

---

### Шаг 3. Пишем маршруты регистрации и входа 🚪

Открой `app.js` и добавь рядом с остальными `require`:

```javascript
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');

// Для простоты оставим секрет в коде. В реальном проекте клади это в .env
const JWT_SECRET = 'dev-secret-change-me';
```

Подключи базу, если ещё не подключал в предыдущей главе:

```javascript
const db = require('./database/db');
```

Теперь добавим маршруты:

```javascript
// Регистрация
app.post('/auth/register', (req, res) => {
  const { email, password } = req.body || {};

  const normalizedEmail = (email || '').trim().toLowerCase();
  const rawPassword = (password || '').trim();

  if (!normalizedEmail.includes('@')) {
    return res.status(400).json({ error: 'Укажи корректный email' });
  }
  if (rawPassword.length < 6) {
    return res.status(400).json({ error: 'Пароль должен быть не короче 6 символов' });
  }

  db.get('SELECT id FROM users WHERE email = ?', [normalizedEmail], (err, row) => {
    if (err) return res.status(500).json({ error: 'Не удалось проверить пользователя' });
    if (row) return res.status(409).json({ error: 'Пользователь с таким email уже существует' });

    const passwordHash = bcrypt.hashSync(rawPassword, 10);
    db.run('INSERT INTO users (email, password_hash) VALUES (?, ?)', [normalizedEmail, passwordHash], function (insertErr) {
      if (insertErr) return res.status(500).json({ error: 'Не удалось создать пользователя' });
      res.status(201).json({ id: this.lastID, email: normalizedEmail });
    });
  });
});

// Вход (логин)
app.post('/auth/login', (req, res) => {
  const { email, password } = req.body || {};
  const normalizedEmail = (email || '').trim().toLowerCase();
  const rawPassword = (password || '').trim();

  if (!normalizedEmail || !rawPassword) {
    return res.status(400).json({ error: 'Укажи email и пароль' });
  }

  db.get('SELECT id, password_hash FROM users WHERE email = ?', [normalizedEmail], (err, user) => {
    if (err) return res.status(500).json({ error: 'Ошибка при поиске пользователя' });
    if (!user) return res.status(401).json({ error: 'Неверный email или пароль' });

    const ok = bcrypt.compareSync(rawPassword, user.password_hash);
    if (!ok) return res.status(401).json({ error: 'Неверный email или пароль' });

    const token = jwt.sign({ userId: user.id }, JWT_SECRET, { expiresIn: '7d' });
    res.json({ token });
  });
});
```

Добавим простую «защиту» для приватных маршрутов:

```javascript
function authenticate(req, res, next) {
  const authHeader = req.headers.authorization || '';
  const token = authHeader.startsWith('Bearer ') ? authHeader.slice(7) : '';

  if (!token) return res.status(401).json({ error: 'Нужен токен авторизации' });

  jwt.verify(token, JWT_SECRET, (err, payload) => {
    if (err) return res.status(401).json({ error: 'Токен недействителен или истёк' });
    req.userId = payload.userId;
    next();
  });
}

// Пример защищённого маршрута: текущий пользователь
app.get('/me', authenticate, (req, res) => {
  db.get('SELECT id, email, created_at FROM users WHERE id = ?', [req.userId], (err, user) => {
    if (err) return res.status(500).json({ error: 'Не удалось получить профиль' });
    if (!user) return res.status(404).json({ error: 'Пользователь не найден' });
    res.json(user);
  });
});
```

Убедись, что эти маршруты стоят до `app.listen(4000, ...)`.

### Шаг 4. Аккуратная обработка ошибок 🧯

Иногда может «что-то сломаться» не по нашей вине. Чтобы сервер отвечал аккуратно, в самом конце `app.js` добавь обработчик непредвиденных ошибок:

```javascript
// Централизованный обработчик ошибок (должен быть В КОНЦЕ всех маршрутов)
app.use((err, req, res, next) => {
  console.error('Unexpected error:', err);
  res.status(500).json({ error: 'Упс! Что-то пошло не так. Попробуй ещё раз.' });
});
```

Советы по коду:
- Для ожидаемых ситуаций возвращай понятные статусы: `400` (неверные данные), `401` (нужна авторизация), `404` (не найдено), `409` (конфликт — пользователь уже есть)
- Текст ошибки делай коротким и человеческим

### Шаг 5. Тестируем в Postman 🧪

Адрес сервера: `http://localhost:4000`

1) Регистрация
- Метод: `POST`
- URL: `http://localhost:4000/auth/register`
- Headers: `Content-Type: application/json`
- Body (raw → JSON):

```json
{ "email": "anna@example.com", "password": "secret123" }
```

Ожидаемо:
```json
{ "id": 1, "email": "anna@example.com" }
```

2) Вход
- Метод: `POST`
- URL: `http://localhost:4000/auth/login`
- Headers: `Content-Type: application/json`
- Body:

```json
{ "email": "anna@example.com", "password": "secret123" }
```

Ожидаемо:
```json
{ "token": "<твой_jwt_токен>" }
```

3) Проверка защищённого маршрута
- Метод: `GET`
- URL: `http://localhost:4000/me`
- Headers: `Authorization: Bearer <твой_jwt_токен>`

Ожидаемо:
```json
{ "id": 1, "email": "anna@example.com", "created_at": "2025-01-01 12:00:00" }
```

4) Проверяем ошибки (полезно!)
- Повтори регистрацию с тем же email → получишь `409` и понятное сообщение
- Введи неверный пароль при входе → `401`
- Удали заголовок `Authorization` при запросе на `/me` → `401`
