## 6. Авторизация через Яндекс 🔐🟡

Цель: пользователь нажимает «Войти через Яндекс», а сервер выдаёт наш JWT (как в главе 4). С этим JWT работают `/me`, `/tasks` и другие защищённые маршруты.

Схема простая:

1. На странице с кнопкой Яндекс проверяет логин и отдаёт временный токен.
2. Страница отправляет этот токен на наш сервер.
3. Сервер узнаёт email пользователя у Яндекса, находит или создаёт запись в SQLite и возвращает наш JWT.

Дополнительные npm-пакеты для этого **не нужны** — хватит того, что уже стоит с главы 4 (`jsonwebtoken`, `bcryptjs`).

---

### Шаг 1. Создаём приложение в Яндексе 🧩

1) Открой [oauth.yandex.ru](https://oauth.yandex.ru) и создай OAuth-приложение.

2) В Redirect URI укажи адрес вспомогательной страницы (см. шаг 4):

```txt
http://127.0.0.1:5500/token.html
```

3) Скопируй `client_id` — он понадобится в `index.html`.

`client_secret` для этой схемы на сервере не используем: токен от Яндекса приходит с фронта после входа пользователя.

---

### Шаг 2. Добавляем `yandex_id` в таблицу `users` 🗄️

Чтобы связать аккаунт Яндекса с нашим пользователем, добавим колонку `yandex_id` — ID пользователя из ответа Яндекса (`profile.id`).

#### Если база уже есть (глава 4)

`CREATE TABLE IF NOT EXISTS` **не меняет** существующую таблицу. Выполни один раз — через DB Browser, `sqlite3 config/db.sqlite` или добавь в `config/db.js`:

```sql
ALTER TABLE users ADD COLUMN yandex_id TEXT;

CREATE UNIQUE INDEX IF NOT EXISTS idx_users_yandex_id ON users(yandex_id);
```

#### Обнови схему в `config/db.js` (для новых установок)

В `CREATE TABLE users` добавь колонку и индекс:

```javascript
db.run(`
  CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email TEXT NOT NULL UNIQUE,
    password TEXT NOT NULL,
    yandex_id TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
  )
`);

db.run(`CREATE UNIQUE INDEX IF NOT EXISTS idx_users_yandex_id ON users(yandex_id)`);
```

Важные моменты:
- **`yandex_id TEXT` без `NOT NULL`** — пользователи, зарегистрированные по email/паролю (глава 4), не имеют Яндекс-аккаунта; колонка остаётся пустой (`NULL`).
- **Уникальный индекс** — один аккаунт Яндекса не может быть привязан к двум пользователям. В SQLite несколько `NULL` в уникальном индексе допустимы.
- В главе 4 колонка с паролем может называться `password` — в примерах ниже используем это имя.

---

### Шаг 3. Две HTML-страницы на фронте 🖥️

Страницы открываем через Live Server (обычно порт `5500`). Важно: везде один и тот же адрес — `http://127.0.0.1:5500`, не `localhost`.

#### `index.html` — страница с кнопкой «Войти через Яндекс»

Подключаем скрипт Яндекса, рисуем кнопку и после входа отправляем токен на сервер:

```html
<!doctype html>
<html lang="ru">
<head>
  <meta charset="utf-8" />
  <script src="https://yastatic.net/s3/passport-sdk/autofill/v1/sdk-suggest-with-polyfills-latest.js"></script>
</head>
<body>
  <div id="container"></div>
  <script>
    window.onload = function () {
      window.YaAuthSuggest.init(
        {
          client_id: 'ВАШ_CLIENT_ID',
          response_type: 'token',
          redirect_uri: 'http://127.0.0.1:5500/token.html'
        },
        'http://127.0.0.1:5500',
        {
          view: 'button',
          parentId: 'container',
          buttonView: 'main',
          buttonTheme: 'light',
          buttonSize: 'm',
          buttonBorderRadius: 0
        }
      )
        .then(function (result) {
          return result.handler();
        })
        .then(async function (data) {
          const response = await fetch('http://127.0.0.1:4000/auth/yandex', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ access_token: data.access_token })
          });

          const result = await response.json();
          console.log('Наш JWT:', result.token);
        })
        .catch(function (error) {
          console.log('Ошибка:', error);
        });
    };
  </script>
</body>
</html>
```

#### `token.html` — короткая служебная страница

Яндекс перенаправляет сюда после входа. Страница забирает токен из адреса и передаёт его обратно на `index.html`:

```html
<!doctype html>
<html lang="ru">
<head>
  <meta charset="utf-8" />
  <script src="https://yastatic.net/s3/passport-sdk/autofill/v1/sdk-suggest-token-with-polyfills-latest.js"></script>
</head>
<body>
  <script>
    window.onload = function () {
      window.YaSendSuggestToken('http://127.0.0.1:5500', { flag: true });
    };
  </script>
</body>
</html>
```

Пользователь эту страницу почти не видит — она мигает на долю секунды.

### Шаг 4. Маршрут на сервере ⚙️

Соберём вход через Яндекс **по частям**. На каждом шаге добавляем одну логику, проверяем в Thunder Client / Postman — и только потом идём дальше.

В конце получится такая схема:

```
страница прислала access_token
  → спросили у Яндекса: кто вошёл?
  → есть yandex_id в базе?       → выдали JWT
  → нет yandex_id, но есть email? → привязали yandex_id, выдали JWT
  → совсем новый?                → создали пользователя, выдали JWT
```

Функция `createTokent` у тебя уже есть в `app.js` (глава 4) — будем её использовать.

---

#### 4.1. Разрешаем запросы с фронта

Страница на `:5500`, сервер на `:4000`. Без этого блока браузер заблокирует запрос:

```javascript
app.use((req, res, next) => {
  res.setHeader('Access-Control-Allow-Origin', 'http://127.0.0.1:5500');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type');
  res.setHeader('Access-Control-Allow-Methods', 'POST, OPTIONS');
  if (req.method === 'OPTIONS') return res.sendStatus(204);
  next();
});
```

---

#### 4.2. Принимаем токен от страницы

Самый простой вариант — убедиться, что токен доходит:

```javascript
app.post('/auth/yandex', (req, res) => {
  const { access_token } = req.body;

  if (!access_token) {
    return res.status(400).json({ error: 'Нет токена от Яндекса' });
  }

  res.json({ ok: true, message: 'Токен получен' });
});
```

**Проверка в Thunder Client:**

- Method: `POST`
- URL: `http://127.0.0.1:4000/auth/yandex`
- Body (JSON):

```json
{
  "access_token": "ваш_токен_от_яндекса"
}
```

> Токен лежит **на верхнем уровне**, не внутри `"data"`.

---

#### 4.3. Спрашиваем у Яндекса, кто вошёл

Странице не доверяем — сами идём в Яндекс:

```javascript
app.post('/auth/yandex', async (req, res) => {
  const { access_token } = req.body;

  const response = await fetch(
    'https://login.yandex.ru/info?format=json&oauth_token=' + access_token
  );
  const data = await response.json();

  res.json({ data }); // смотрим, что вернул Яндекс
});
```

**Проверка:** в ответе увидишь объект с полями `id`, `default_email`, `real_name` и др.

Нам нужны только два:

| Поле Яндекса | Зачем |
|---|---|
| `id` | сохраним в `yandex_id` |
| `default_email` | почта пользователя |

---

#### 4.4. Достаём `id` и `default_email`

```javascript
app.post('/auth/yandex', async (req, res) => {
  const { access_token } = req.body;

  const response = await fetch(
    'https://login.yandex.ru/info?format=json&oauth_token=' + access_token
  );
  const data = await response.json();
  const { id, default_email } = data;

  res.json({ id, default_email });
});
```

**Проверка:** в ответе — два значения, без лишнего.

---

#### 4.5. Ищем пользователя по `yandex_id`

Человек уже входил через Яндекс? Смотрим в базе:

```javascript
db.get('SELECT id, email FROM users WHERE yandex_id = ?', [String(id)], (err, row) => {
  if (err) return res.status(500).json({ error: err.message });

  res.json({ row }); // row = { id, email } или пусто
});
```

**Проверка:**

| `row` | Значение |
|---|---|
| `{ id: 1, email: "..." }` | Уже входил через Яндекс |
| пусто | В базе нет такого `yandex_id` |

> `row.id` — это id **нашего** пользователя в SQLite, не id Яндекса.

---

#### 4.6. Если `yandex_id` есть — выдаём JWT

Нашли — человек уже с нами. Создаём пропуск и отвечаем:

```javascript
if (row) {
  const jwt = createTokent(row.id, row.email);

  return res.json({
    jwt,
    data: {
      id: row.id,
      email: row.email
    }
  });
}
```

**Проверка:** повторный вход тем же Яндекс-аккаунтом возвращает JWT без создания новой записи.

---

#### 4.7. Если `yandex_id` нет — ищем по email

Может, человек регистрировался по почте/паролю (глава 4), а Яндекс впервые:

```javascript
db.get('SELECT id, email FROM users WHERE email = ?', [default_email], (err, row) => {
  if (err) return res.status(500).json({ error: err.message });

  res.json({ row }); // смотрим, есть ли почта
});
```

**Проверка:** если регистрировался через `/auth/register` — `row` не пустой.

---

#### 4.8. Почта есть — привязываем `yandex_id`

Записываем `yandex_id` к существующему пользователю:

```javascript
if (row) {
  const userId = row.id;

  db.run(
    'UPDATE users SET yandex_id = ? WHERE id = ?',
    [String(id), userId],
    (err) => {
      if (err) return res.status(500).json({ error: err.message });

      const jwt = createTokent(userId, default_email);

      res.json({
        success: 'Привязали Яндекс к существующему пользователю',
        jwt,
        user: {
          id: userId,
          email: default_email,
          yandex_id: id
        }
      });
    }
  );
}
```

**Проверка:** после этого в базе у старого пользователя появился `yandex_id`.

---

#### 4.9. Почты нет — создаём нового

Совсем новый человек:

```javascript
db.run(
  'INSERT INTO users (email, password, yandex_id) VALUES (?, ?, ?)',
  [default_email, '0', String(id)],
  function (err) {
    if (err) return res.status(500).json({ error: err.message });

    const jwt = createTokent(this.lastID, default_email);

    res.status(201).json({
      message: 'Пользователь успешно создан',
      jwt,
      user: {
        id: this.lastID,
        email: default_email,
        yandex_id: id
      }
    });
  }
);
```

> `this.lastID` — id только что созданной записи. Берём его **внутри** колбэка `db.run`.

**Проверка:** в таблице `users` появилась новая строка.

---

#### 4.10. Собираем всё вместе

Финальный маршрут — три ветки внутри вложенных `db.get`:

```javascript
app.post('/auth/yandex', async (req, res) => {
  const { access_token } = req.body;

  const response = await fetch(
    'https://login.yandex.ru/info?format=json&oauth_token=' + access_token
  );
  const data = await response.json();
  const { id, default_email } = data;

  // 1. Уже входил через Яндекс?
  db.get(
    'SELECT id, email FROM users WHERE yandex_id = ?',
    [String(id)],
    (err, row) => {
      if (err) return res.status(500).json({ error: err.message });

      if (row) {
        const jwt = createTokent(row.id, row.email);
        return res.json({
          jwt,
          data: { id: row.id, email: row.email }
        });
      }

      // 2. Есть аккаунт с такой почтой?
      db.get(
        'SELECT id, email FROM users WHERE email = ?',
        [default_email],
        (err2, rowByEmail) => {
          if (err2) return res.status(500).json({ error: err2.message });

          if (rowByEmail) {
            const userId = rowByEmail.id;

            db.run(
              'UPDATE users SET yandex_id = ? WHERE id = ?',
              [String(id), userId],
              (err3) => {
                if (err3) return res.status(500).json({ error: err3.message });

                const jwt = createTokent(userId, default_email);
                return res.json({
                  success: 'Привязали Яндекс к существующему пользователю',
                  jwt,
                  user: { id: userId, email: default_email, yandex_id: id }
                });
              }
            );
            return;
          }

          // 3. Совсем новый
          db.run(
            'INSERT INTO users (email, password, yandex_id) VALUES (?, ?, ?)',
            [default_email, '0', String(id)],
            function (err4) {
              if (err4) return res.status(500).json({ error: err4.message });

              const jwt = createTokent(this.lastID, default_email);
              res.status(201).json({
                message: 'Пользователь успешно создан',
                jwt,
                user: {
                  id: this.lastID,
                  email: default_email,
                  yandex_id: id
                }
              });
            }
          );
        }
      );
    }
  );
});
```

**Порядок веток важен:**

1. Сначала `yandex_id` — точное совпадение, быстрый вход.
2. Потом `email` — связываем старый аккаунт с Яндексом.
3. И только потом `INSERT` — новый пользователь.

**Частая ошибка:** `SQLITE_CONSTRAINT` при повторном `INSERT` — значит, пользователь уже есть, а проверка выше не сработала. Проверь, что `row` / `rowByEmail` читаешь **до** вставки.

### Шаг 5. Проверяем 🧪

1) Запусти сервер: `npm run dev` (порт `4000`).

2) Открой `index.html` через Live Server: `http://127.0.0.1:5500/index.html`

3) Нажми «Войти через Яндекс» и разреши доступ.

4) В консоли браузера (F12) должен появиться наш JWT.

5) Проверь в Postman:
- `GET http://127.0.0.1:4000/me`
- заголовок `Authorization: Bearer <jwt>`
