## 6. Авторизация через Яндекс (OAuth2) 🔐🟡

Цель: дать пользователю войти через Яндекс и при успешном логине выдавать наш `JWT` (как в главе 4), чтобы дальше работали защищённые маршруты (`/me`, `GET /tasks` и т.д.).

Будем использовать `passport` + стратегию `passport-yandex`, а после обратного вызова Яндекса — создадим/найдём пользователя в SQLite и подпишем ему JWT.

---

### Шаг 1. Создаём приложение в Яндекс и получаем `client_id` / `client_secret` 🧩

1) Открой Яндекс-консоль разработчика и создай OAuth-приложение.
2) Укажи Redirect URI (Callback URL) в формате:

```txt
http://localhost:4000/auth/yandex/callback
```

3) Скопируй:
- `client_id`
- `client_secret`

4) Для выдачи email в приложении запросим scope `login:email` (ниже).

---

### Шаг 2. Ставим нужные пакеты 📦

В проекте (где `app.js`) выполни:

```bash
npm i passport passport-yandex jsonwebtoken bcryptjs
```

Почему `bcryptjs` тоже нужен:
- мы сохраняем пользователю в SQLite поле `password_hash` (как в главе 4), даже если логин идёт через OAuth. Для “OAuth-пользователей” кладём случайный bcrypt-хэш, чтобы не ломать схему.

---

### Шаг 3. Добавляем `yandex_id` в таблицу `users` 🗄️

В главе 4 у тебя таблица `users` хранит `email` и `password_hash`.
Если в твоём проекте колонка с паролем называется иначе (например, `password`), замени в примерах `password_hash` на актуальное имя.

Чтобы связать Яндекс-аккаунт с пользователем в нашей базе, добавим колонку `yandex_id`:

```sql
ALTER TABLE users ADD COLUMN yandex_id TEXT;

-- Чтобы один Яндекс-аккаунт соответствовал только одному пользователю:
CREATE UNIQUE INDEX IF NOT EXISTS idx_users_yandex_id ON users(yandex_id);
```

---

### Шаг 4. Подключаем Passport и настраиваем стратегию Яндекса ⚙️

В `app.js` подключи Passport и настрой стратегию.

Пример (адаптируй пути/импорты под свою структуру, а также используй свой `JWT_SECRET`):

```javascript
const passport = require('passport');
const YandexStrategy = require('passport-yandex');
const jwt = require('jsonwebtoken');
const bcrypt = require('bcryptjs');
const crypto = require('crypto');

const db = require('./config/db'); // если у тебя db подключен иначе — поправь

const JWT_SECRET = 'dev-secret-change-me';

passport.use(new YandexStrategy(
  {
    clientID: 'YANDEX_CLIENT_ID',
    clientSecret: 'YANDEX_CLIENT_SECRET',
    callbackURL: 'http://localhost:4000/auth/yandex/callback',
  },
  function (accessToken, refreshToken, profile, done) {
    const yandexId = profile.id;
    const email = (profile.emails && profile.emails[0] && profile.emails[0].value) ? profile.emails[0].value : null;

    if (!email) {
      return done(new Error('Не удалось получить email из Яндекса'));
    }

    // 1) Сначала пробуем найти по yandex_id
    db.get('SELECT id, email FROM users WHERE yandex_id = ?', [yandexId], (err, userRow) => {
      if (err) return done(err);

      if (userRow) return done(null, userRow);

      // 2) Если yandex_id не найден, пробуем найти по email (на случай, если пользователь уже был создан через регистрацию)
      db.get('SELECT id, email FROM users WHERE email = ?', [email], (err2, userByEmail) => {
        if (err2) return done(err2);

        if (userByEmail) {
          // Привязываем yandex_id к уже существующему пользователю
          db.run('UPDATE users SET yandex_id = ? WHERE id = ?', [yandexId, userByEmail.id], function (updateErr) {
            if (updateErr) return done(updateErr);
            done(null, userByEmail);
          });
          return;
        }

        // 3) Иначе создаём пользователя “под OAuth”
        const randomPassword = crypto.randomBytes(32).toString('hex');
        const passwordHash = bcrypt.hashSync(randomPassword, 10);

        db.run(
          'INSERT INTO users (email, password_hash, yandex_id) VALUES (?, ?, ?)',
          [email, passwordHash, yandexId],
          function (insertErr) {
            if (insertErr) return done(insertErr);
            done(null, { id: this.lastID, email });
          }
        );
      });
    });
  }
));

// ВАЖНО: чтобы Passport работал, нужна инициализация
app.use(passport.initialize());
```

Поля в `profile`:
- `profile.id` — уникальный идентификатор пользователя Яндекса
- `profile.emails[0].value` — email (при scope `login:email`)

### Шаг 5. Добавляем роуты входа через Яндекс и выдачу JWT 🚪

В `app.js` добавь два маршрута:

1) `/auth/yandex` — редирект на Яндекс для логина
2) `/auth/yandex/callback` — обработка ответа Яндекса и выдача JWT

Пример:

```javascript
app.get(
  '/auth/yandex',
  passport.authenticate('yandex', { scope: ['login:email'], session: false })
);

app.get(
  '/auth/yandex/callback',
  passport.authenticate('yandex', { failureRedirect: '/auth/yandex', session: false }),
  (req, res) => {
    // req.user вернётся из verify callback (done(null, user))
    const token = jwt.sign(
      { userId: req.user.id },
      JWT_SECRET,
      { expiresIn: '7d' }
    );

    // Для API-формата удобнее вернуть JSON, а не редиректить на фронт:
    res.json({ token });
  }
);
```

### Шаг 6. Проверяем в браузере и Postman 🧪

1) Открой в браузере:

```txt
http://localhost:4000/auth/yandex
```

2) Разреши доступ Яндексу.
3) Сервер должен ответить JSON:

```json
{ "token": "<jwt>" }
```

4) Проверь защищённый маршрут (как в главе 4), например `/me`:
- `GET http://localhost:4000/me`
- Header `Authorization: Bearer <jwt>`
