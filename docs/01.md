# 1. Введение и основы Express.js 🚀

## Что такое Express.js? 🍽️
Представь, что ты хочешь открыть ресторан. Тебе нужно:

- Кухня 🍳 (где готовить) = Сервер (где обрабатываются данные)
- Официант 🧑‍🍳 (который принимает заказы) = Express.js (который принимает запросы)
- Меню 📋 (что можно заказать) = API (что можно запросить)

Express.js — это как официант в ресторане. Он принимает заказы от посетителей (запросы от браузера) и приносит им еду (ответы).

## Зачем нужен Express.js? 💡
- Простота — легко создавать веб‑приложения
- Гибкость — можно делать что угодно
- Популярность — легко найти помощь

## Установка Node.js ⬇️

### Шаг 1: Скачивание Node.js 📥
1. Перейди на сайт: [nodejs.org](https://nodejs.org)
2. Скачай версию **LTS** (Long Term Support) — она стабильнее, меньше конфликтов
3. Запусти установщик и следуй инструкциям

### Шаг 2: Проверка установки ✅

Открой командную строку (терминал) и выполни:

```bash
node -v
```

Ты должен увидеть что-то вроде: `v22.18.0`.

Затем выполни:

```bash
npm -v
```

Ты должен увидеть что-то вроде: `v11.5.3`.

**Если видишь версию — всё установлено верно!**

## Создание проекта 🛠️
### Шаг 3: Создание папки проекта 📁
1. Создай новую папку для проекта в любом удобном месте с названием `app_server` (без пробелов, на английском языке)
2. Открой эту папку в **Visual Studio Code**
3. Создай файл `package.json` при помощи команды:
```bash
npm init -y
```

### Что такое package.json? 🧾
Представь, что `package.json` — это паспорт твоего проекта. Как паспорт содержит всю важную информацию о человеке, так и `package.json` содержит ключевые сведения о проекте.

### Зачем нужен package.json?
- ℹ️ Информация о проекте — название, версия, описание
- 📦 Список зависимостей — какие библиотеки использует проект
- 🧰 Скрипты — команды для запуска, тестирования, сборки
- 👤 Автор и лицензия — кто создал проект и на каких условиях
- ⚙️ Настройки — конфигурация для различных инструментов

### Шаг 4: Установка Express.js 📦
Для установки Express.js выполни следующую команду:
```bash
npm i express
```
После установки пакетов твой `package.json` автоматически обновится:
```json
{
...
    "dependencies": {
        "express": "^5.1.1"
    }
...
}
```
Также появится файл `package-lock.json` — с точными версиями пакетов, и директория `node_modules/` — папка с установленными пакетами.

### Что такое dependencies? 📚

`dependencies` — это список библиотек, которые нужны твоему проекту для работы в `production`. Это как список ингредиентов для рецепта:

- `^5.1.1` — версия пакета, означающая: >= 5.1.1 и < 6.0.0
- `express` — название пакета
- `5.1.1` — точная версия

Помимо `dependencies` существуют `devDependencies` — пакеты, нужные только для разработки, и `peerDependencies` — пакеты, которые должны быть установлены отдельно.
