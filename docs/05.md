## 5. Личные задачи: доступ только владельцу 👤✅

Цель: привязать задачи к пользователю. Каждый видит и меняет только свои задачи. Добавим колонку в таблицу, обновим маршруты и проверим всё в Postman.

---

### Шаг 1. Обновляем таблицу `tasks` (добавляем `user_id`) 🛠️

Нам нужна связь «задача → пользователь». В SQLite это просто: добавим колонку `user_id` и (по желанию) индекс.

Выполни один раз (через код или консоль для SQLite):

```sql
ALTER TABLE tasks ADD COLUMN user_id INTEGER; -- новая колонка, по умолчанию NULL
```

Опционально сделай индекс (ускорит выборку по пользователю):

```sql
CREATE INDEX IF NOT EXISTS idx_tasks_user_id ON tasks(user_id);
```

Важные моменты:
- Нельзя добавить `NOT NULL` без значения по умолчанию — поэтому просто добавляем колонку и контролируем всё в коде.
- Если у тебя уже есть старые задачи, им проставится `NULL` в `user_id`. Можно:
  - удалить старые тестовые задачи; или
  - привязать их к своему аккаунту, узнав свой `id` и выполнив:

```sql
UPDATE tasks SET user_id = <твой_user_id> WHERE user_id IS NULL;
```

### Шаг 2. Создаём задачи «от имени» текущего пользователя ➕

Убедись, что из главы 4 у тебя есть `authenticate` и `req.userId`. Теперь изменим маршруты задач в `app.js`.

1) Создание задачи: теперь записываем `user_id`

```javascript
// POST /tasks — создать задачу текущему пользователю
app.post('/tasks', authenticate, (req, res) => {
  const { title } = req.body || {};
  const cleanTitle = (title || '').trim();
  if (!cleanTitle) {
    return res.status(400).json({ error: 'Укажи непустой title' });
  }

  const sql = 'INSERT INTO tasks (title, user_id) VALUES (?, ?)';
  db.run(sql, [cleanTitle, req.userId], function (err) {
    if (err) return res.status(500).json({ error: 'Не удалось создать задачу' });
    res.status(201).json({ id: this.lastID, title: cleanTitle, done: 0 });
  });
});
```

2) Список задач: показываем только свои

```javascript
// GET /tasks — список только своих задач
app.get('/tasks', authenticate, (req, res) => {
  const sql = 'SELECT id, title, done, created_at FROM tasks WHERE user_id = ? ORDER BY id DESC';
  db.all(sql, [req.userId], (err, rows) => {
    if (err) return res.status(500).json({ error: 'Не удалось получить список задач' });
    res.json(rows);
  });
});
```

3) Получить одну задачу: проверяем «чужая/своя»

```javascript
// GET /tasks/:id — получить задачу, если она твоя
app.get('/tasks/:id', authenticate, (req, res) => {
  const { id } = req.params;
  const sql = 'SELECT id, title, done, created_at FROM tasks WHERE id = ? AND user_id = ?';
  db.get(sql, [id, req.userId], (err, row) => {
    if (err) return res.status(500).json({ error: 'Не удалось получить задачу' });
    if (!row) return res.status(404).json({ error: 'Задача не найдена' });
    res.json(row);
  });
});
```

4) Обновить задачу: разрешаем менять только свою

```javascript
// PUT /tasks/:id — изменить title и/или done, если задача твоя
app.put('/tasks/:id', authenticate, (req, res) => {
  const { id } = req.params;
  const { title, done } = req.body;

  if (!title) return res.status(400).json({ error: "Поле title не найдено" });
  if (!done) return res.status(400).json({ error: "Поле done не найдено" });

  const sql = `UPDATE tasks SET title = ?, done = ? WHERE id = ? AND user_id = ?`;

  db.run(sql, [title, done, id, req.user.id], (err) => {
    if (err) return res.status(500).json({ error: "Не удалось обновить задачу" });
    res.json({ success: "Данные успешно обновлены" });
  });
});
```

5) Удалить задачу: тоже только свою

```javascript
// DELETE /tasks/:id — удалить задачу, если она твоя
app.delete('/tasks/:id', authenticate, (req, res) => {
  const { id } = req.params;
  const sql = 'DELETE FROM tasks WHERE id = ? AND user_id = ?';
  db.run(sql, [id, req.userId], function (err) {
    if (err) return res.status(500).json({ error: 'Не удалось удалить задачу' });
    if (this.changes === 0) return res.status(404).json({ error: 'Задача не найдена' });
    res.json({ success: true });
  });
});
```

Совет: на «чужие» ресурсы лучше отвечать `404` — так ты не раскрываешь, что задача существует.

### Шаг 3. Обработка ошибок — коротко и по делу 🧯

- `400` — некорректные данные (пустой `title`, нечего обновлять)
- `401` — нет токена или он неверный
- `404` — задача не найдена или не принадлежит пользователю
- `500` — непредвиденная ошибка базы/сервера

Централизованный обработчик из главы 4 оставляем в конце `app.js`.

### Шаг 4. Проверяем в Postman 🧪

Адрес: `http://localhost:4000`

1) Войти и получить токен
- `POST /auth/login` → получи `token`

2) Создать задачу
- `POST /tasks`
- Headers: `Authorization: Bearer <token>`, `Content-Type: application/json`
- Body:

```json
{ "title": "Выучить Express" }
```

3) Список своих задач
- `GET /tasks` (с тем же `Authorization`)

4) Обновить задачу
- `PUT /tasks/1`
- Body (любой из полей):

```json
{ "title": "Повторить Express", "done": true }
```

5) Удалить задачу
- `DELETE /tasks/1`

6) Проверка изоляции
- Зарегистрируй второго пользователя, залогинься → увидишь пустой список (`GET /tasks`)
- Попробуй открыть/изменить `id` задачи первого пользователя → `404`

### Шаг 5. (Необязательно) Жёсткая связь в самой базе 🔗

Быстрый вариант выше — всё контролирует приложение. Если хочешь «жёстко» в базе, то в SQLite внешние ключи лучше задать при создании таблицы. Для уже существующей таблицы чаще делают так:

1) Создать новую таблицу с внешним ключом

```sql
PRAGMA foreign_keys = ON;

CREATE TABLE tasks_new (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  done INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  user_id INTEGER NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

2) Перенести данные (только те строки, где `user_id` заполнен)

```sql
INSERT INTO tasks_new (id, title, done, created_at, user_id)
SELECT id, title, done, created_at, user_id FROM tasks WHERE user_id IS NOT NULL;
```

3) Переименовать таблицы в транзакции

```sql
BEGIN TRANSACTION;
DROP TABLE tasks;
ALTER TABLE tasks_new RENAME TO tasks;
COMMIT;
```

Далее оставь код из шагов выше — он уже использует `user_id`.
